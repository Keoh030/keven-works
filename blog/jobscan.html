Systeme die
funktionieren.

Wie ich JOBSCAN, Whisper und eine Claude API in einem Nachmittag zum Laufen gebracht habe. Und was dabei fast alles schiefgelaufen ist.

Heute hat JOBSCAN zum ersten Mal funktioniert.

Das klingt simpel. Ist es aber nicht. Hinter diesem Satz stecken ein falscher API Key, ein Evaluation-Account ohne Credits, ein kaputtes Alias im Finder, und die Erkenntnis dass ein API Key nach dem Erstellen nur einmal angezeigt wird. Danach ist er weg.

Aber er läuft. Und das ist alles was zählt.

JOBSCAN ist mein KI-Bewerbungsassistent. Er läuft direkt im Browser, ohne Backend, ohne Server. Du gibst eine Stellenanzeige rein. Er analysiert sie mit Claude, schreibt dir ein passendes Anschreiben, gibt dir eine Match-Bewertung, zeigt dir was du betonen solltest.

Das Prinzip dahinter ist einfach: Claude API direkt aus dem Browser ansprechen. Kein Umweg über eigene Server. Kein monatlicher Hosting-Cost. Eine HTML-Datei, ein API Key, fertig.

Nicht cool aussehen. Funktionieren.

Das ist der Unterschied zwischen einem Tool das ich zeige und einem Tool das ich benutze. JOBSCAN ist beides. Aber heute war der erste Tag wo ich es wirklich benutzt habe.

Was ich dabei gelernt habe:

API Keys nie ein zweites Mal abrufbar. Du siehst ihn einmal, direkt nach dem Erstellen. Danach ist er weg. Ich habe zwei Keys gebraucht um das zu verstehen.

Evaluation Access ≠ echter Zugang. Anthropic gibt dir einen kostenlosen Account zum Testen. Aber ohne Credits funktioniert die API nicht. 5$ reichen für den Start. Den Unterschied merkt man erst wenn der erste Call mit 401 zurückkommt.

Kaputte Alias-Dateien im Finder können trotzdem schreiben. Mein Website-Ordner war als Alias verknüpft und sah kaputt aus. Aber die Dateien die ich über den Pfad geschrieben habe landeten trotzdem am richtigen Ort. Manchmal funktionieren Dinge ohne dass man es sieht.

Parallel dazu: Whisper installiert.

Whisper ist ein Open-Source Spracherkennungsmodell von OpenAI. Es läuft lokal auf meinem Mac, kostet nichts pro Nutzung, und erkennt Deutsch erstaunlich gut. Installation über pip, ffmpeg als Abhängigkeit, sox für die Mikrofon-Aufnahme.

Was das bedeutet: Ich kann jetzt sprechen statt tippen. 5 Minuten frei reden → Whisper transkribiert → Claude strukturiert → Blog-Post-Entwurf. Das ist die Pipeline die ich aufbaue.

Heute läuft noch nicht alles automatisch. Aber die Teile sind da. Whisper. JOBSCAN. Claude. Blog. Session-Log.

Systeme bauen bedeutet nicht, dass alles sofort läuft. Es bedeutet, dass die richtigen Teile vorhanden sind.

Morgen ist MST Kickoff. Mein erstes bezahltes Projekt startet. Ich bekomme die Zugangsdaten, richte das Backup ein, starte den Security Audit.

Das ist kein großer Moment von außen. Keine Bühne. Kein Applaus. Nur ein Termin, eine Checkliste, und die erste Anzahlung.

Aber genau so fühlt sich echter Aufbau an.

Nicht warten. Bauen.
